## 核心功能

### Chats（對話）

**Chats** 是 Claude Desktop 的基本對話介面。

**主要特色：**
- 每次與 Claude 的交流都會建立一個獨立對話
- 對話會自動儲存，可以隨時回來繼續討論
- 適合用於：一般性提問、快速諮詢、測試想法

**實用技巧：**
- 使用「搜尋和參考過去對話」功能，讓 Claude 記住之前討論過的內容

---

### Projects（專案）

**Projects** 提供更有組織性的工作空間，適合處理需要大量背景資料的專案。

**主要特色：**
- 可以上傳相關文件（教材、程式碼、文件等）作為背景知識
- 可以設定專案說明（Custom Instructions），讓 Claude 理解專案的特定脈絡
- 專案內的所有對話都能存取這些共享資源

**實際操作：**  
[Projects 實際操作指南](./Projects/README.md)

---

### Artifacts（工件）

**Artifacts** 是 Claude 生成的獨立內容視窗，用於展示和編輯生成的實質內容。當 Claude 建立程式碼、文件、圖表等內容時，會自動顯示在右側的 Artifacts 面板，而不是只塞在聊天訊息中。

**核心概念：**

Artifacts 將對話與作品區分開來：
- **左側聊天區**：與 Claude 的對話互動
- **右側作品區**：專注展示和編輯生成內容

就像在迷你 IDE 或文件編輯器中工作，這些內容可以獨立開啟、專心編輯，並反覆讓 Claude 協助調整。

**主要特色：**

**即時編輯與迭代**
- 可以直接在 Artifacts 中修改 React 元件、HTML、Markdown、Mermaid 圖表等內容
- 修改後可以再請 Claude 根據您的調整繼續優化
- 支援多種格式：React 元件、HTML、Markdown、Mermaid 圖表等

**清晰展示**
- 上課或展示時，左側保留對話記錄，右側專注展示程式碼或圖表
- 學生不會被對話記錄干擾，更容易專注於實際內容
- 適合即時示範和互動教學

**教學應用：**

**課堂示範流程**
1. 讓 Claude 先產生範例程式或圖表
2. 使用 Artifacts 直接展示「修改程式碼、重構、修復 bug」的完整過程
3. 學生可以清楚看到每個修改步驟和效果

**活教材應用**
- 將同一份 Artifacts 當作「活教材」
- 一邊修改一邊詢問 Claude，示範人機協作的工作流程
- 學生可以觀察 AI 如何根據回饋持續改進內容

## 程式設計功能

### Code（程式碼編輯）

**Code** 是整合在桌面應用程式中的 AI 程式設計助手。

**主要功能：**

**程式碼庫操作**
- 可以讀取和修改電腦上的整個程式碼專案（支援最多 200,000 個 token 的上下文）
- 能夠跨多個檔案執行自動化的程式設計任務

**自主執行任務**
- 自動進行除錯（debugging）
- 程式碼重構（refactoring）
- 測試撰寫
- 文件生成
- Bug 修復

**多會話並行作業**
- 可以同時執行多個 Claude Code 會話
- 例如：一個會話處理 bug 修復，另一個進行程式碼重構，第三個更新文件
- 每個會話都有獨立的 Git worktree，互不干擾

**工具整合**
- 與 Git 完整整合
- 可使用命令列工具
- 支援 MCP (Model Context Protocol) 伺服器連接

**應用價值：**
- 可以展示 AI 如何協助實際的程式開發流程
- 學生可以看到從需求到實作的完整過程
- 配合「反向解構」教學法，讓學生觀察 AI 如何組織和撰寫程式碼

**注意事項：**  
需要付費訂閱（Max、Pro、Team 或 Enterprise）才能使用此功能。

### Cowork（協作模式）

**Cowork** 是 Claude 在 2026 年 1 月推出的新功能，目前處於「研究預覽」階段。它將 Claude Code 的強大能力帶給非程式設計師使用。

**核心特色：**

**🤖 自主執行任務**
- 您只需要描述想要的結果，Claude 就能自動規劃並執行多步驟的工作
- 例如：「整理我的下載資料夾，按類型分類並重新命名」
- Claude 會制定計畫、執行，然後呈現完成的成果

**📁 直接存取本地檔案**
- 您可以指定一個資料夾讓 Claude 存取
- Claude 能在該資料夾中讀取、編輯、創建檔案
- 所有操作都在沙盒環境中進行，確保安全性

**⚡ 實際應用場景**
- 整理雜亂的檔案（分類、重新命名）
- 從一堆收據截圖中建立費用報表
- 從零散筆記中撰寫完整報告
- 製作簡報、試算表等文件
- 結合 Claude in Chrome 還能執行需要瀏覽器的任務

**使用限制（目前）：**
- ⚠️ **僅限 macOS** 的 Claude Desktop 應用程式
- ⚠️ 只對 **Pro 和 Max 訂閱用戶**開放
- ⚠️ 不支援 Projects、記憶功能、分享功能
- ⚠️ 必須保持應用程式開啟，否則任務會中斷

**與一般聊天的差異：**

| 一般 Claude 對話 | Cowork 模式 |
|----------------|------------|
| 您問→它答 | 您委託→它完成 |

就像真的把工作交給同事一樣，您可以排隊多個任務，讓 Claude 平行處理，完成後再回來檢查成果。

**教學應用：**  
這個功能特別適合自動化重複性的檔案整理、文件準備等工作。

## 擴展功能

### Connectors（連接器）

**Connectors（連接器）** 是一個讓 Claude 能夠連接到外部工具和數據源的功能系統。

**核心功能：**

**連接器目錄**
- 這是一個精選的工具集合，您可以一鍵瀏覽和連接
- 所有連接器都基於 **MCP（Model Context Protocol）** 建立，這是 Anthropic 開發的開放標準

**兩大類型連接器：**

1. **網頁連接器**（遠端服務）
   - 服務：Notion、Canva、Stripe、Linear、Slack 等
   - 適用於：Claude 網頁版、桌面版和手機版（需付費方案）

2. **桌面擴展**（本地應用程式）
   - 服務：Figma、Socket、Prisma 等
   - 適用於：所有 Claude Desktop 用戶（包含免費用戶）

**實際應用價值：**

透過連接器，Claude 可以：
- 直接存取您的工具和數據
- 提供更相關和具體的回應
- 在您的工具中直接協作

**範例：**  
原本您只能問「寫發布說明」得到範本，但連接 Linear 後，您可以直接要求「從 Linear 撰寫最新 sprint 的發布說明」，Claude 會抓取實際的工單資料來生成。

**如何使用：**

在 Claude 介面中：
- 點擊左下角的「搜尋和工具」按鈕
- 或到「設定 > 連接器」
- 瀏覽目錄並選擇要連接的工具
- 通過 OAuth 認證授權

**進階功能：**  
您也可以建立**自訂連接器**來連接未在目錄中的服務（Pro、Max、Team、Enterprise 方案）。

**教學應用：**  
連接 GitHub、Linear 等開發工具，讓 Claude 直接存取專案資料來協助課程準備或學生專案指導。

### Memory（記憶功能）

**Memory（記憶功能）** 讓 Claude 能夠記住您在過去對話中分享的資訊，並在未來的對話中自然地應用這些記憶。

**核心特點：**

**自動記憶生成**
- Claude 會從您的對話歷史中自動提取重要資訊
- 這些記憶會定期在背景更新
- 最近的對話可能還需要一些時間才會反映在記憶中

**自然應用**
- Claude 會根據對話的相關性選擇性地應用記憶
- 不會每次都提醒「我記得...」，而是像人類同事一樣自然地運用這些資訊
- 對於一般性問題可能不會應用記憶，但對個人化請求會充分利用

**實際範例：**

假設您之前告訴 Claude：
- 您是程式教師
- 您教成人轉職者
- 您使用「逆向拆解」教學法

之後當您問「如何設計課程」時，Claude 會自然地根據這些背景提供適合成人學習者和您教學法的建議，而不是給出通用的教學建議。

**管理記憶：**

您可以透過以下方式管理：

1. **查看記憶**：要求 Claude 顯示它記住了什麼
2. **新增記憶**：明確告訴 Claude 要記住某些事情
3. **刪除記憶**：要求 Claude 忘記特定資訊
4. **更新記憶**：修正或更新已存在的記憶

**指令範例：**
- 「請記住我現在住在台北」
- 「請忘記關於我工作的資訊」
- 「我不再在那家公司工作了」

**隱私設計：**
- 在無痕對話模式中，記憶功能會被停用
- 當您刪除對話時，從那些對話中衍生的資訊會在每晚清除
- 這是您在 Claude 設定中可以開啟或關閉的功能之一

**應用價值：**  
Claude 可以記住您的教學方法、學生背景、常用工具等，讓每次對話都更有針對性和效率。

### Skills（技能）

**Skills（技能）** 是 Claude 在執行特定任務時可以參考的專業知識庫，特別是在使用電腦功能（如創建文件、簡報等）時。

**什麼是 Skills？**

Skills 本質上是「最佳實踐資料夾」，包含：
- 經過大量測試和改進的操作指南
- 針對特定任務類型的詳細說明
- 幫助 Claude 產出高品質成果的技巧

**目前可用的 Skills：**

**文件處理類**
- **docx**：Word 文件的創建、編輯、追蹤變更
- **xlsx**：Excel 試算表處理、公式、資料分析
- **pptx**：PowerPoint 簡報製作
- **pdf**：PDF 檔案處理、表單填寫

**開發設計類**
- **frontend-design**：創建高品質的前端介面和網頁組件

**系統知識類**
- **product-self-knowledge**：Anthropic 產品的權威資訊

**進階功能**
- **skill-creator**：教您如何創建自己的 skills

**工作原理：**

當您要求 Claude 執行任務時（例如「製作一份簡報」），Claude 會：
1. 先讀取相關的 skill 文件（如 `pptx/SKILL.md`）
2. 依照裡面的最佳實踐來執行任務
3. 產出更專業、更符合規範的結果

**自訂 Skills：**

您可以創建**使用者自訂 skills**：
- 儲存在 `/mnt/skills/user/` 目錄
- 針對您特定的工作流程或需求
- 例如：如果您有特定的課程教材格式，可以創建一個 skill 來指導 Claude 如何製作符合您標準的教材

**對教學的應用：**

作為程式教師，您可以：
- 使用現有 skills 來快速生成教材（如 docx、pptx）
- 創建自己的 skill 來標準化教學文件的產出
- 例如：創建一個「程式教學講義」skill，定義您希望的程式碼範例格式、說明結構等

這就像給 Claude 一本「操作手冊」，確保它以您期望的方式完成任務，特別適合需要重複執行且有特定品質要求的工作。

### Extension/MCP（擴展功能）

Claude Desktop 的 **Extension** 主要指的是 **MCP (Model Context Protocol) 伺服器**。

**MCP 伺服器的功能：**

MCP 伺服器讓 Claude Desktop 能夠：

1. **連接外部工具和服務**
   - Google Drive、Slack、GitHub 等
   - 本地檔案系統
   - 資料庫和 API

2. **擴展 Claude 的能力**
   - 讀取和搜尋您的文件
   - 執行程式碼
   - 查詢資料庫
   - 自動化工作流程

**管理 MCP 伺服器：**

您可以在 Claude Desktop 的設定檔中管理這些擴充功能：
- **Mac**：`~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**：`%APPDATA%\Claude\claude_desktop_config.json`

**常見的 MCP 伺服器：**

- **檔案系統存取**：讀取本地檔案
- **Google Drive**：存取雲端文件
- **Git**：程式碼版本控制
- **資料庫連接**：查詢 SQL 資料

**與 Connectors 的差異：**

- **Connectors**：主要用於連接第三方服務（如 Notion、Linear），通常透過 UI 介面設定
- **MCP 伺服器**：更底層的擴展機制，需要手動配置設定檔，適合開發者和進階用戶
