# 範例 2：會議記錄整理與摘要（工作）

## 目標

將會議討論內容整理成結構化的會議記錄，學習如何將雜亂資訊轉換為專業文件。

## 教學重點

- 學習如何將雜亂資訊結構化
- 掌握專業文件格式
- 提升資訊整理能力

## 操作步驟與 Prompt 對話範例

### 步驟 1：提供會議內容並產生標準格式

**請複製以下 prompt 貼到 Claude 對話框（記得替換成您實際的會議內容）：**

```
以下是一場產品開發會議的討論內容，請幫我整理成標準化的會議記錄，並在 Artifacts 中展示。

會議內容：
[在此貼上您的會議內容，例如：]
今天的會議主要討論 Q4 產品功能規劃。產品經理提到要新增使用者儀表板功能，預計 10 月底完成。技術團隊表示需要評估 API 整合的複雜度，下週三會給出技術方案。設計師建議參考競品的介面設計，並提出需要 2 週時間完成 UI 設計稿。最後決定由小明負責前端開發，小華負責後端 API，專案經理由小李擔任。

會議記錄格式要求：
1. 會議基本資訊（日期、時間、參與人員）
2. 討論主題
3. 重要決議
4. 行動項目（包含負責人和截止日期）
5. 待討論事項

請使用 Artifacts 產生格式清晰的 Markdown 文件。
```

---

### 步驟 2：補充會議基本資訊

**如果您的會議內容沒有包含基本資訊，可以使用以下 prompt：**

```
會議時間：2024 年 3 月 15 日 14:00-15:30
會議地點：線上會議室
主持人：王經理
參與人員：產品經理（張三）、技術主管（李四）、設計師（王五）、專案經理（趙六）

請將這些資訊加入 Artifacts 的會議記錄中，並保持格式一致。
```

---

### 步驟 3：完善重點事項和決議

**在 Artifacts 中檢視 Claude 產生的會議記錄後，可以直接編輯或要求補充：**

**Prompt：**

```
請檢查並完善 Artifacts 中的會議記錄，確保：
1. 所有重要決議都有清楚記錄
2. 時間點和里程碑明確標示
3. 決議背後的考量或原因有適當說明
4. 格式清晰易讀，重點突出
```

---

### 步驟 4：補充行動項目和負責人

**請複製以下 prompt 確保行動項目完整：**

```
請檢查 Artifacts 中的會議記錄，確保行動項目（Action Items）包含：
1. 每個行動項目的明確描述
2. 負責人姓名
3. 預定完成日期或時程
4. 優先順序（如：高、中、低）
5. 需要協作的項目標示清楚

如果會議內容中有提到但未明確說明的行動項目，請合理推斷並加入。
```

---

### 步驟 5：調整格式和內容

**根據需求調整格式，可以使用以下 prompt：**

```
請優化 Artifacts 中的會議記錄格式：
1. 使用表格呈現行動項目，欄位包含：項目、負責人、截止日期、狀態
2. 重要決議使用粗體或列表強調
3. 加入會議記錄的版本資訊（第幾次修訂）
4. 在文件末尾加入「下次會議日期」欄位

請保持專業且易讀的格式。
```

## 進階應用範例

### 範例 A：從多個來源整理會議記錄

**如果您有多份筆記或討論串，可以使用以下 prompt：**

```
我提供了三份不同來源的會議相關內容，請整合成一份完整的會議記錄，在 Artifacts 中展示：

來源 1：[貼上第一份內容]
來源 2：[貼上第二份內容]
來源 3：[貼上第三份內容]

請：
1. 去除重複資訊
2. 按時間順序或主題分類組織內容
3. 標示資訊來源（如需要）
4. 產出格式化的會議記錄
```

---

### 範例 B：多語言會議記錄

**如果是多語言會議，可以要求翻譯或雙語版本：**

```
會議主要用英文進行，但參與者包含中文使用者。請在 Artifacts 中產生：
1. 中文版的會議記錄（主要版本）
2. 重要專有名詞保留英文
3. 行動項目同時提供中英文對照

請確保專業術語翻譯準確。
```

---

### 範例 C：加入視覺化元素

**可以使用 Mermaid 圖表呈現專案時程：**

```
請在 Artifacts 的會議記錄中加入專案時程圖，使用 Mermaid 語法繪製甘特圖，顯示：
1. 各項任務的開始和結束時間
2. 負責人資訊
3. 任務之間的相依關係

例如：
- 使用者儀表板設計（2 週，設計師）
- API 開發（3 週，後端工程師）
- 前端開發（2 週，前端工程師，需等待設計完成）

請整合到會議記錄文件中。
```

## 範例模板：完整的會議記錄 Prompt

**如果您想要一次完成，可以使用這個完整的 prompt 範本：**

```
請幫我整理以下會議內容為標準化的會議記錄，在 Artifacts 中展示 Markdown 格式文件。

【會議資訊】
- 日期：2024 年 3 月 15 日
- 時間：14:00-15:30
- 地點：線上會議室
- 主持人：[主持人姓名]
- 參與人員：[列出所有參與者]

【會議討論內容】
[在此貼上完整的會議討論內容或筆記]

【格式要求】
1. 會議記錄標題和基本資訊
2. 討論主題（分項列出）
3. 重要決議（清楚標示）
4. 行動項目表格（項目、負責人、截止日期、優先順序、狀態）
5. 待討論事項或需要後續追蹤的項目
6. 下次會議預定時間（如有）

請使用清晰的分層標題和列表，讓文件易於閱讀和追蹤。
```

---

## 使用提示

1. **提供完整資訊**：盡量提供完整的會議內容，包括參與者、時間、討論重點等
2. **逐步完善**：先產生基本格式，再逐步補充細節
3. **直接編輯**：在 Artifacts 中可以直接編輯內容，補充遺漏的資訊
4. **格式一致性**：建立一套固定的格式模板，方便後續使用
5. **行動項目追蹤**：確保每個行動項目都有明確的負責人和時程
