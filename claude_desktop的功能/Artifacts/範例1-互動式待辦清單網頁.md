# 範例 1：建立互動式待辦清單網頁（程式）

## 目標

使用 HTML、CSS 和 JavaScript 建立一個功能完整的待辦清單應用，透過 Artifacts 學習前端開發的完整流程。

## 教學重點

- 展示前端開發的完整流程
- 示範如何逐步迭代功能
- 學習程式碼結構和最佳實踐

## 操作步驟與 Prompt 對話範例

### 步驟 1：建立基本 HTML 結構和樣式

**請複製以下 prompt 貼到 Claude 對話框：**

```
請幫我建立一個互動式待辦清單（To-Do List）的網頁應用程式。使用 HTML、CSS 和 JavaScript，並在 Artifacts 中展示。

需求：
1. 包含一個輸入框和「新增」按鈕，可以新增待辦事項
2. 顯示所有待辦事項的清單
3. 每項待辦事項旁邊要有「完成」和「刪除」按鈕
4. 完成的項目要有視覺標記（如刪除線）
5. 使用現代化的 CSS 樣式，讓介面美觀易用

請使用 Artifacts 產生完整的 HTML 檔案。
```

---

### 步驟 2：在 Artifacts 中檢視和修改程式碼

當 Claude 產生程式碼後，您可以：

1. **檢視 Artifacts 中的程式碼**：點擊右側 Artifacts 區域，查看完整程式碼
2. **嘗試修改**：直接編輯 HTML 或 CSS，例如改變顏色或字體大小

**如果要在對話中要求修改樣式，可使用以下 prompt：**

```
請修改 Artifacts 中的樣式，將主題色改為藍色系，並且：
- 增加按鈕的圓角和陰影效果
- 完成的待辦事項使用淡灰色背景
- 優化手機版的顯示效果
```

---

### 步驟 3：加入新功能

**請複製以下 prompt 加入新功能：**

```
請在 Artifacts 的待辦清單中加入以下功能：
1. 計數器：顯示「總共 X 項，已完成 Y 項」
2. 清除已完成項目：新增一個「清除已完成」按鈕
3. 本地儲存：使用 localStorage 儲存待辦事項，重新整理頁面後不會消失

請直接在 Artifacts 中更新程式碼。
```

---

### 步驟 4：優化使用者體驗

**請複製以下 prompt 優化互動效果：**

```
請優化待辦清單的使用者體驗：
1. 在輸入框按 Enter 鍵也可以新增項目
2. 加入動畫效果：新增或刪除項目時有淡入淡出效果
3. 如果沒有待辦事項，顯示提示訊息「還沒有待辦事項，新增一個吧！」
4. 加入輸入驗證：不允許空字串或只有空白的待辦事項

請更新 Artifacts 中的程式碼。
```

---

### 步驟 5：除錯和優化

**如果遇到問題，可以使用以下 prompt 除錯：**

```
我發現待辦清單有以下問題：
[請描述您遇到的問題，例如：刪除按鈕無法運作、樣式顯示不正常等]

請檢查 Artifacts 中的程式碼並修正錯誤。
```

**或者要求程式碼優化：**

```
請檢查並優化 Artifacts 中的程式碼：
1. 檢查是否有重複的程式碼可以重構
2. 確保程式碼符合最佳實踐
3. 加入註解說明關鍵功能
4. 優化效能，確保操作流暢
```

---

## 進階延伸

完成基本功能後，可以嘗試以下進階功能：

**Prompt：**

```
請在待辦清單中加入以下進階功能：
1. 優先順序：可以設定每項待辦事項的優先度（高、中、低）
2. 分類標籤：可以為待辦事項加上標籤（如「工作」、「個人」、「購物」）
3. 排序功能：可以依建立時間、優先順序或字母順序排序
4. 編輯功能：點擊待辦事項可以編輯內容

請在 Artifacts 中實作這些功能。
```

---

## 使用提示

1. **逐步迭代**：一次只要求一個或兩個功能，確保每步都運作正常再繼續
2. **直接編輯**：在 Artifacts 中可以直接修改程式碼，修改後可以要求 Claude 檢查或優化
3. **學習理解**：每次 Claude 產生程式碼後，試著理解每段程式碼的作用
4. **實驗修改**：大膽嘗試修改樣式或功能，學習程式設計的最佳方式就是動手做
