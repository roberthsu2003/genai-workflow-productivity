# 範例 3：專案企劃書撰寫（工作）

## 目標

建立一份完整的專案企劃書，包含背景、目標、執行方案和預算，學習結構化寫作和企劃思維。

## 教學重點

- 學習企劃書的標準格式
- 培養邏輯思考和結構化寫作能力
- 了解如何整合多種內容類型（文字、表格、圖表）

## 操作步驟與 Prompt 對話範例

### 步驟 1：產生企劃書大綱

**請複製以下 prompt 貼到 Claude 對話框：**

```
我想建立一份「公司官方網站改版專案」的企劃書。請先幫我產生企劃書的大綱結構，並在 Artifacts 中展示。

專案背景：
公司現有網站已經運作 5 年，技術架構過時，使用者體驗不佳。需要進行全面改版，提升品牌形象和用戶體驗。

請產生的企劃書應包含以下章節：
1. 專案背景與目標
2. 現況分析
3. 專案執行方案
4. 時程規劃
5. 預算評估
6. 風險評估與因應措施
7. 預期效益

請在 Artifacts 中建立 Markdown 格式的企劃書大綱。
```

---

### 步驟 2：完善各章節內容

**逐項完善內容，先從背景開始：**

**Prompt：**

```
請完善 Artifacts 中企劃書的「專案背景與目標」章節，內容應包括：
1. 公司現況說明
2. 現有網站存在的問題（具體描述）
3. 改版的主要目標（可量化）
4. 專案的核心價值主張

請用專業且具體的文字撰寫，每段約 100-150 字。
```

---

### 步驟 3：加入數據分析

**補充數據和具體資訊：**

**Prompt：**

```
請在 Artifacts 的企劃書中加入以下數據分析：
1. 現況分析章節中加入：
   - 網站流量統計（如：每月 10,000 訪客）
   - 使用者反饋問題統計（列出前 5 大問題）
   - 競品網站功能比較表
2. 預期效益中加入可量化的指標：
   - 預期流量成長 30%
   - 使用者停留時間增加 50%
   - 轉換率提升 20%

請使用表格呈現數據比較。
```

---

### 步驟 4：建立時程規劃

**使用表格或圖表規劃時程：**

**Prompt：**

```
請在 Artifacts 的企劃書中完善「時程規劃」章節，使用表格呈現：

專案時程規劃（共 4 個月）：
1. 第一階段：需求分析與設計（4 週）
   - 第 1-2 週：使用者研究與需求訪談
   - 第 3 週：資訊架構設計
   - 第 4 週：UI/UX 設計稿
2. 第二階段：開發實作（6 週）
   - 前端開發（3 週）
   - 後端開發（3 週）
3. 第三階段：測試與優化（3 週）
4. 第四階段：上線與推廣（1 週）

請用表格或甘特圖形式呈現，並標示關鍵里程碑。
```

---

### 步驟 5：加入 Mermaid 流程圖

**視覺化呈現專案流程：**

**Prompt：**

```
請在 Artifacts 的企劃書「專案執行方案」章節中，使用 Mermaid 語法加入專案執行流程圖，顯示：
1. 專案啟動
2. 需求分析
3. 設計階段
4. 開發階段
5. 測試階段
6. 上線部署
7. 後續維護

並標示各階段的主要交付項目和決策點。

請將流程圖整合到 Markdown 文件中。
```

---

### 步驟 6：補充預算和風險評估

**完善預算規劃：**

**Prompt：**

```
請在 Artifacts 的企劃書中加入詳細的「預算評估」和「風險評估與因應措施」：

預算項目包含：
- 人力成本：設計師 2 人 × 4 個月、工程師 3 人 × 3 個月
- 外部資源：主機、CDN、第三方服務
- 行銷推廣：SEO 優化、廣告投放
- 其他雜項

風險評估應包括：
1. 時程延遲風險及因應措施
2. 技術難題風險及備案
3. 預算超支風險及控制方式
4. 人員異動風險及替代方案

請用表格清楚呈現。
```

## 完整範例：一次產生企劃書

**如果您想一次完成，可以使用這個完整的 prompt：**

```
請幫我撰寫一份「新產品開發專案」的完整企劃書，在 Artifacts 中展示 Markdown 格式文件。

【專案資訊】
- 專案名稱：智慧健康管理 APP 開發專案
- 專案期間：6 個月
- 預算上限：500 萬元

【專案背景】
公司希望開發一款整合健康數據追蹤、運動記錄、營養建議的 APP，目標族群為 25-45 歲的上班族。

【企劃書應包含】
1. 專案背景與市場分析
2. 專案目標與成功指標
3. 產品功能規劃（核心功能與進階功能）
4. 技術架構與開發方案
5. 專案時程規劃（使用表格或甘特圖）
6. 團隊組織與分工
7. 預算明細表（人力、設備、行銷等）
8. 風險評估與因應策略（至少 5 項風險）
9. 預期效益與 ROI 分析
10. 附錄（如有參考資料或市場調查）

【格式要求】
- 使用清晰的章節標題
- 數據使用表格呈現
- 時程規劃使用甘特圖（Mermaid）
- 重要內容使用列表或粗體強調
- 保持專業且易讀的格式

請產出一份完整、專業的企劃書。
```

## 進階應用

### 加入 SWOT 分析

**Prompt：**

```
請在 Artifacts 的企劃書中加入 SWOT 分析表格（優勢、劣勢、機會、威脅），針對「智慧健康管理 APP 開發專案」進行分析。

請用表格形式呈現，並針對每項提出具體說明。
```

---

### 加入競爭分析

**Prompt：**

```
請在「現況分析」章節中加入競爭對手分析表，比較 3-5 個主要競品的：
- 核心功能
- 價格策略
- 市場佔有率
- 優缺點

請用表格呈現，並分析我們的差異化優勢。
```

---

### 加入財務預測

**Prompt：**

```
請在企劃書中加入「財務預測與 ROI 分析」章節，包含：
1. 預估開發成本（月費）
2. 預估營收模式（訂閱費、廣告收入等）
3. 3 年內的財務預測表
4. ROI 計算
5. 損益平衡點分析

請用表格和簡單的計算說明呈現。
```

---

## 使用提示

1. **先有大綱再填內容**：先建立清晰的大綱結構，再逐步完善各章節
2. **數據具體化**：盡量使用具體數據和可量化的指標
3. **視覺化輔助**：善用表格、圖表（Mermaid）讓企劃書更清晰
4. **邏輯連貫**：確保各章節之間邏輯連貫，背景→目標→方案→效益一氣呵成
5. **直接編輯**：在 Artifacts 中可以直接編輯和完善內容，靈活調整格式
