# Claude Skills（技能）

**Skills（技能）** 是一組可重複使用的「文件夾 + 指令 + 資源」模組，幫助 Claude 在特定工作情境中表現更一致、效率更高。Skills 就像給 Claude 一本「操作手冊」，確保它以您期望的方式完成任務。

---

## 什麼是 Skills？

### 核心概念

Skills 是一組包含以下內容的模組：
- **指令文件**（`SKILL.md`）：定義如何使用這個 skill、最佳實踐、格式要求
- **範例文件**：示範預期的輸出格式和內容
- **樣板檔案**：可重複使用的模板（如簡報樣板、文件格式）
- **腳本和資源**：輔助工具和參考資料

### Skills 的作用

1. **自動辨識**：Claude 會自動辨識您的請求是否需要使用某個 skill
2. **一致性**：確保每次執行相同任務時，產出結果格式一致
3. **效率提升**：減少重複說明，直接使用已定義的流程和格式
4. **品質保證**：依照最佳實踐產出高品質的結果

### 與其他功能的比較

| 功能 | 用途 | 適用場景 |
|------|------|----------|
| **Skills** | 定義工作流程、格式標準、最佳實踐 | 重複性任務、需要一致格式的工作 |
| **Projects** | 組織相關文件和對話 | 長期專案、需要背景資料的任務 |
| **Custom Instructions** | 設定 Claude 的個人偏好和風格 | 個人化設定、通用偏好 |
| **Prompts** | 單次對話的指令 | 一次性任務、臨時需求 |

**Skills 特別適合：**
- 製作格式固定的文件（簡報、報告、講義）
- 執行重複性的工作流程
- 需要遵守品牌樣式或格式規範的任務
- 標準化教學材料的產出

## Skills 的類型

### 1. 內建 Skills（Anthropic 提供）

Anthropic 提供的預設 skills，包含：

**文件處理類**
- **docx**：Word 文件的創建、編輯、追蹤變更
- **xlsx**：Excel 試算表處理、公式、資料分析
- **pptx**：PowerPoint 簡報製作
- **pdf**：PDF 檔案處理、表單填寫

**開發設計類**
- **frontend-design**：創建高品質的前端介面和網頁組件

**系統知識類**
- **product-self-knowledge**：Anthropic 產品的權威資訊

**進階功能**
- **skill-creator**：教您如何創建自己的 skills

### 2. 自訂 Skills（使用者建立）

您可以建立自己的 skills：
- 儲存在 `/mnt/skills/user/` 目錄
- 針對您特定的工作流程或需求
- 例如：課程講義格式、公司品牌樣式、個人工作流程

### 3. 組織 Skills（Team/Enterprise）

- 由組織管理員建立和分享
- 團隊成員可以共用
- 確保團隊使用統一的格式和流程

## 工作原理

### 自動辨識與使用

當您要求 Claude 執行任務時（例如「製作一份簡報」），Claude 會：

1. **自動辨識**：判斷您的請求是否需要使用某個 skill
2. **載入 Skill**：如果合適，自動讀取相關的 skill 文件（如 `pptx/SKILL.md`）
3. **依照規範執行**：依照 skill 中定義的最佳實踐、格式要求來執行任務
4. **產出結果**：產出更專業、更符合規範的結果

### 實際範例

**未使用 Skills 時：**
- 您問：「幫我製作一份簡報」
- Claude 回應：使用通用的簡報格式，可能不符合您的需求，每次格式可能不一致

**使用 Skills 後：**
- 您問：「幫我製作一份簡報」
- Claude 回應：自動讀取 `pptx/SKILL.md`，依照最佳實踐製作符合專業標準的簡報，格式一致且高品質

**明確指定 Skill：**
- 您問：「請使用 pptx skill 幫我製作一份簡報」
- Claude 回應：明確使用 pptx skill，確保完全依照 skill 的規範執行

## Skills 的結構

### 基本結構

一個 Skill 通常包含以下內容：

```
SkillName/
  ├── SKILL.md          # 主要指令文件（必需）
  ├── examples/         # 範例文件（可選）
  │   ├── example1.docx
  │   └── example2.pptx
  ├── templates/        # 樣板檔案（可選）
  │   ├── template1.pptx
  │   └── template2.docx
  └── resources/        # 其他資源（可選）
      ├── scripts/
      └── configs/
```

### SKILL.md 的基本格式

```markdown
# Skill 名稱

## 說明
這個 skill 的用途和適用場景

## 何時使用
說明什麼情況下應該使用這個 skill

## 如何使用
1. 步驟一
2. 步驟二
3. 步驟三

## 格式要求
- 格式要求 1
- 格式要求 2

## 最佳實踐
- 實踐 1
- 實踐 2
```

### 自訂 Skills 的儲存位置

- **使用者自訂 Skills**：`/mnt/skills/user/` 目錄
- **組織 Skills**：由管理員管理（Team/Enterprise）
- **內建 Skills**：系統預設位置（不可修改）

## 對教學的應用

作為程式教師，您可以：

### 使用現有 Skills

- 使用現有 skills 來快速生成教材（如 docx、pptx）
- 確保產出的文件符合專業標準

### 創建自訂 Skills

- 創建自己的 skill 來標準化教學文件的產出
- 例如：創建一個「程式教學講義」skill，定義您希望的程式碼範例格式、說明結構等
- 確保每次生成的教材都符合您的教學風格和需求

### 實際應用場景

1. **課程講義製作**
   - 使用 `docx` skill 生成標準化的課程講義
   - 自訂 skill 定義講義的格式和結構

2. **簡報製作**
   - 使用 `pptx` skill 製作專業簡報
   - 確保簡報符合教學需求

3. **試算表分析**
   - 使用 `xlsx` skill 處理學生成績資料
   - 自動生成分析報告

## 如何啟用 Skills

### 前置條件

使用 Skills 功能需要滿足以下條件：

1. **訂閱方案**
   - 需要 **Pro、Max、Team 或 Enterprise** 訂閱方案
   - 免費版不支援 Skills 功能

2. **啟用必要功能**
   - **Code Execution**（程式碼執行）：必須開啟
   - **File Creation**（檔案創建）：必須開啟
   - 這些功能可以在 `設定` → `Capabilities` 中啟用

3. **Skills 功能**
   - 在 `設定` → `Capabilities` → `Skills` 中確認已開啟

### 設定步驟

1. **開啟 Claude Desktop 設定**
   - 點擊左下角的「設定」圖示
   - 選擇 `Capabilities`

2. **啟用必要功能**
   - 確認 **Code Execution** 已開啟
   - 確認 **File Creation** 已開啟
   - 確認 **Skills** 已開啟

3. **驗證功能**
   在對話中測試：
   ```
   請告訴我目前有哪些可用的 skills？
   ```
   如果 Claude 能夠列出 skills，表示功能正常。

## 常見問題與 Debug

### 常見問題

#### Q1：Claude 沒有使用 skill 怎麼辦？

**問題描述：**
當您要求 Claude 執行任務時，Claude 可能沒有自動使用 skill。

**可能原因：**
1. 沒有明確指定使用 skill
2. Skill 的描述不夠清楚
3. 請求的內容與 skill 的適用場景不符

**解決方法：**
1. **明確指定 skill**
   ```
   請使用 pptx skill 幫我製作簡報
   ```

2. **檢查 skill 是否啟用**
   - 前往 `設定` → `Capabilities` → `Skills`
   - 確認 Skills 功能已開啟

3. **驗證 skill 是否可用**
   ```
   請告訴我目前有哪些可用的 skills？
   ```

4. **檢查前置條件**
   - 確認 Code Execution 已開啟
   - 確認 File Creation 已開啟
   - 確認訂閱方案支援 Skills（Pro/Max/Team/Enterprise）

---

#### Q2：Skill 產出的結果不符合預期怎麼辦？

**問題描述：**
Claude 使用了 skill，但產出的結果不符合您的需求。

**可能原因：**
1. 需求描述不夠清楚
2. Skill 的定義不夠具體
3. 沒有提供足夠的範例或參考

**解決方法：**
1. **提供更詳細的需求**
   - 明確說明格式要求
   - 提供具體的範例
   - 說明特殊要求

2. **迭代改進**
   ```
   請根據以下要求修改這份文件：
   - 要求 1
   - 要求 2
   - 要求 3
   ```

3. **檢查 skill 內容**
   - 如果是自訂 skill，檢查 SKILL.md 的描述是否清楚
   - 確認 skill 的適用場景是否正確

---

#### Q3：如何確認 Claude 是否使用了 skill？

**方法 1：直接詢問**
```
請告訴我你使用了哪些 skills 來完成這個任務？
```

**方法 2：觀察回應**
- Claude 通常會在回應中提到它參考了 skill
- 查看回應中是否有提到 skill 的名稱或相關內容

**方法 3：檢查產出結果**
- 如果產出結果符合 skill 定義的格式和標準，通常表示 skill 已被使用

---

#### Q4：自訂 skill 沒有生效怎麼辦？

**問題描述：**
建立了自訂 skill，但 Claude 沒有使用它。

**可能原因：**
1. Skill 檔案位置不正確
2. SKILL.md 格式錯誤
3. Skill 描述不夠清楚

**解決方法：**
1. **檢查檔案位置**
   - 確認 skill 儲存在 `/mnt/skills/user/` 目錄
   - 確認資料夾名稱和檔案名稱正確

2. **檢查 SKILL.md 格式**
   - 確認檔案是有效的 Markdown 格式
   - 確認包含必要的描述和說明

3. **明確指定 skill**
   ```
   請使用 [skill名稱] skill 來完成這個任務
   ```

4. **重新載入 skill**
   - 重新啟動 Claude Desktop
   - 或明確要求 Claude 重新讀取 skill

---

#### Q5：可以同時使用多個 skills 嗎？

**答案：**
可以！Claude 可以同時使用多個 skills，但需要明確指定：

```
請使用 pptx skill 和 xlsx skill 幫我：
1. 製作一份簡報（使用 pptx skill）
2. 分析數據並生成圖表（使用 xlsx skill）
```

### Debug 技巧

#### 技巧 1：逐步測試

1. **先測試基本功能**
   ```
   請告訴我目前有哪些可用的 skills？
   ```

2. **測試特定 skill**
   ```
   請使用 [skill名稱] skill 完成一個簡單的測試任務
   ```

3. **逐步增加複雜度**
   - 先完成簡單任務
   - 再逐步增加需求複雜度

#### 技巧 2：檢查 skill 內容

如果是自訂 skill，可以要求 Claude 讀取 skill 內容：

```
請讀取 /mnt/skills/user/[skill名稱]/SKILL.md 的內容，
並告訴我這個 skill 的用途和使用方法。
```

#### 技巧 3：對比測試

比較使用 skill 和不使用 skill 的差異：

1. **不使用 skill**
   ```
   幫我製作一份簡報
   ```

2. **使用 skill**
   ```
   請使用 pptx skill 幫我製作一份簡報
   ```

3. **比較結果**
   - 觀察兩者的差異
   - 確認 skill 是否發揮作用

### 錯誤訊息處理

#### 錯誤 1：「Skills 功能未啟用」

**解決方法：**
1. 前往 `設定` → `Capabilities` → `Skills`
2. 確認 Skills 功能已開啟
3. 確認 Code Execution 和 File Creation 已開啟

#### 錯誤 2：「找不到指定的 skill」

**解決方法：**
1. 確認 skill 名稱正確
2. 檢查 skill 是否存在
3. 確認 skill 的儲存位置正確

#### 錯誤 3：「訂閱方案不支援 Skills」

**解決方法：**
1. 確認您的訂閱方案是 Pro、Max、Team 或 Enterprise
2. 免費版不支援 Skills 功能
3. 需要升級訂閱方案
