# 案例 4：自訂程式教學講義 Skill

本案例展示如何從零開始創建一個自訂的 **程式教學講義 skill**，完全符合您的教學風格和需求。

---

## 📋 假設情境：「建立專屬的程式教學講義格式」

### 背景設定

**使用者：** 張老師（程式設計教師）
**使用工具：** Claude Desktop + 自訂 Skill
**需求：** 
- 希望建立一個專屬的程式教學講義 skill
- 這個 skill 要定義：
  - 講義的標準格式和結構
  - 程式碼範例的格式要求
  - 說明文字的組織方式
  - 練習題的格式
  - 視覺元素的使用規範

**目標：**
- 從零開始建立一個完整的 skill
- 定義所有格式和內容要求
- 確保每次使用都能產出符合標準的講義

---

## 🎯 自訂 Skill 的實際應用

### 未建立自訂 Skill 的情況

**你：** 「請幫我製作程式教學講義」

**Claude 回應：** 
會使用通用的格式製作講義，但可能不符合您的教學風格和特定需求。

---

### 建立自訂 Skill 後的情況

**你：** 「請使用我的程式教學講義 skill 製作講義」

**Claude 回應：**
會使用您自訂的 skill，產出的講義完全符合您的教學風格和格式要求。

---

## 💡 實際操作步驟

### 第一步：規劃 Skill 的內容

在建立 skill 之前，先規劃好 skill 應該包含哪些內容：

1. **Skill 的基本資訊**
   - Skill 名稱
   - Skill 說明
   - 適用場景

2. **格式要求**
   - 文件結構
   - 標題層級
   - 段落格式
   - 列表格式

3. **內容要求**
   - 程式碼範例格式
   - 說明文字格式
   - 練習題格式
   - 視覺元素使用

4. **最佳實踐**
   - 內容組織原則
   - 教學技巧
   - 常見錯誤避免

---

### 第二步：建立 Skill

#### **對話範例 1：建立 Skill 資料夾和檔案**

**你：**
```
請幫我在 /mnt/skills/user/ 目錄下建立一個新的 skill，
名稱為 "programming-teaching-material"。

請建立：
1. 資料夾：/mnt/skills/user/programming-teaching-material/
2. 檔案：SKILL.md
```

**Claude 的回應：**
會：
1. 建立 skill 資料夾
2. 建立 SKILL.md 檔案
3. 確認檔案已建立

---

#### **對話範例 2：撰寫 Skill 內容**

**你：**
```
請幫我撰寫 "programming-teaching-material" skill 的完整內容。

這個 skill 用於製作程式設計課程的教學講義，需要包含以下內容：

【Skill 基本資訊】
- 名稱：程式教學講義製作 Skill
- 說明：用於製作標準化的程式設計課程講義
- 適用場景：程式設計課程、程式教學、技術文件

【文件結構要求】
1. 封面頁
   - 課程名稱（大標題）
   - 週次和主題（副標題）
   - 授課教師（小字）
   - 日期

2. 目錄
   - 自動生成
   - 包含所有章節

3. 課程介紹
   - 本週課程概述
   - 學習重點

4. 學習目標
   - 使用編號列表
   - 每個目標要具體明確

5. 主要內容
   - 使用階層式標題（標題 1、標題 2、標題 3）
   - 每個章節包含：
     * 概念說明
     * 程式碼範例
     * 執行結果
     * 重點整理

6. 程式碼範例格式
   - 使用程式碼區塊
   - 包含詳細註解
   - 註解格式：
     * 單行註解：使用 #
     * 多行註解：使用 """ """
   - 關鍵部分要有說明

7. 練習題
   - 基礎題（至少 3 題）
   - 進階題（至少 2 題）
   - 每題包含：
     * 題目說明
     * 提示（可選）
     * 解答

8. 總結
   - 本週重點回顧
   - 下週預告

【格式要求】
- 標題使用階層式結構
- 段落之間要有適當間距
- 列表使用項目符號或編號
- 程式碼使用等寬字體
- 重要概念使用粗體或強調

【內容組織原則】
1. 由淺入深：從簡單概念開始，逐步深入
2. 理論與實作並重：每個概念都要有對應的程式碼範例
3. 互動性：包含練習題和思考題
4. 視覺輔助：適當使用圖表、流程圖說明

【教學技巧】
1. 使用具體範例說明抽象概念
2. 提供常見錯誤和解決方法
3. 連結實際應用場景
4. 鼓勵學生動手實作

請撰寫完整的 SKILL.md 內容。
```

**Claude 的回應：**
會：
1. 根據您的需求撰寫 skill 內容
2. 組織成結構化的 Markdown 文件
3. 儲存在指定的位置

---

#### **對話範例 3：測試自訂 Skill**

**你：**
```
請使用我剛才建立的 "programming-teaching-material" skill 
來製作一份 Python 基礎程式設計的第一週講義。

課程資訊：
- 課程名稱：Python 基礎程式設計
- 週次：第 1 週
- 主題：Python 環境設定與基本語法
- 授課教師：張老師

請確認講義是否符合 skill 中定義的所有要求。
```

**Claude 的回應：**
會：
1. 讀取您自訂的 skill
2. 依照 skill 的要求製作講義
3. 確認所有格式要求都已符合

## ⚠️ 注意事項

### 建立 Skill 的技巧

1. **明確的需求定義**
   - 在建立 skill 前，先明確定義所有需求
   - 使用具體的範例說明

2. **結構化組織**
   - 使用清晰的章節結構
   - 每個部分都要有明確的說明

3. **測試和迭代**
   - 建立 skill 後要測試
   - 根據測試結果調整 skill 內容

### 常見問題

**Q：Skill 應該儲存在哪裡？**

A：使用者自訂的 skill 應該儲存在 `/mnt/skills/user/` 目錄下。

**Q：Skill 檔案應該使用什麼格式？**

A：Skill 檔案通常是 Markdown 格式（`.md`），檔名為 `SKILL.md`。

**Q：如何確認 skill 是否生效？**

A：使用 skill 製作講義後，檢查是否符合 skill 中定義的要求。

**Q：可以參考現有的 skill 嗎？**

A：可以！您可以先查看現有的 skill，了解 skill 的結構和格式。
