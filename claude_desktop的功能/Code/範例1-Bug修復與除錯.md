# 範例 1：Bug 修復與除錯

## 目標

使用 Claude Code 來診斷和修復程式碼中的錯誤，學習系統化的除錯流程。

## 教學重點

- 展示 AI 如何協助診斷程式錯誤
- 學習如何描述問題和提供上下文
- 了解除錯的最佳實踐

## 操作步驟與 Prompt 對話範例

### Prompt 範例 1：描述錯誤並請求診斷

**請複製以下 prompt 貼到 Claude Code 對話框：**

```
我的 Python 專案中有一個計算平均成績的函式出現錯誤。

專案位置：[請指定您的專案路徑，例如：/Users/username/myproject]

錯誤訊息：
"TypeError: unsupported operand type(s) for +: 'int' and 'str'"

問題描述：
當我執行 calculate_average() 函式時，傳入包含整數和字串的列表會出現這個錯誤。
我只想計算數字的部分，但不知道如何正確處理。

請幫我：
1. 找出問題所在的檔案和程式碼
2. 診斷錯誤的原因
3. 提供修復方案
4. 確保修復後的程式碼能正確處理混合類型的列表
```

**使用情境說明：**
- 這個 prompt 展示了如何向 AI 清楚描述問題
- 包含錯誤訊息、問題描述和期望的結果
- 讓學生學習如何有效溝通技術問題

---

### Prompt 範例 2：系統化檢查多個相關問題

**請複製以下 prompt 貼到 Claude Code 對話框：**

```
我的專案中似乎有多個地方都有類似的資料驗證問題。

專案位置：[請指定您的專案路徑]

問題：
1. 使用者輸入驗證不一致
2. 某些函式沒有檢查空值（None）
3. 錯誤處理不夠完善

請幫我：
1. 掃描整個專案，找出所有相關的驗證和錯誤處理問題
2. 建立一份問題清單
3. 逐一修復這些問題
4. 確保修復後的程式碼有一致的錯誤處理模式

修復原則：
- 使用統一的驗證函式
- 提供清楚的錯誤訊息
- 不要中斷整個程式的執行
```

**使用情境說明：**
- 這個 prompt 展示了如何要求 AI 進行系統化的檢查
- 不僅修復單一問題，而是改善整體程式碼品質
- 讓學生了解除錯不只是修復錯誤，更是改善程式碼的機會

---

## 教學建議

### 課堂示範流程

1. **展示問題診斷過程**
   - 開啟一個有錯誤的專案
   - 使用 Prompt 範例 1，讓學生觀察 AI 如何分析錯誤
   - 強調描述問題的清晰度很重要

2. **示範修復過程**
   - 讓 AI 提出修復方案
   - 討論不同修復方式的優缺點
   - 驗證修復後的程式碼

3. **系統化改善**
   - 使用 Prompt 範例 2，展示如何處理多個相關問題
   - 強調程式碼一致性的重要性
   - 討論如何建立可維護的程式碼

### 學習重點

✅ **問題描述技巧**：學習如何清楚描述技術問題  
✅ **系統化思考**：不只修復單一錯誤，而是考慮整體程式碼品質  
✅ **程式碼審查**：觀察 AI 如何分析和改進程式碼  
✅ **最佳實踐**：了解錯誤處理和驗證的標準做法

---

> 💡 **提示**：學生可以根據自己的專案調整 prompt 中的路徑和問題描述，直接複製貼上使用。
