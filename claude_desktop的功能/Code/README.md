# Code（程式碼編輯）實際操作指南

**Code** 是整合在桌面應用程式中的 AI 程式設計助手。

## 主要功能

### 程式碼庫操作
- 可以讀取和修改電腦上的整個程式碼專案（支援最多 200,000 個 token 的上下文）
- 能夠跨多個檔案執行自動化的程式設計任務

### 自主執行任務
- 自動進行除錯（debugging）
- 程式碼重構（refactoring）
- 測試撰寫
- 文件生成
- Bug 修復

### 多會話並行作業
- 可以同時執行多個 Claude Code 會話
- 例如：一個會話處理 bug 修復，另一個進行程式碼重構，第三個更新文件
- 每個會話都有獨立的 Git worktree，互不干擾

### 工具整合
- 與 Git 完整整合
- 可使用命令列工具
- 支援 MCP (Model Context Protocol) 伺服器連接

## 應用價值
- 可以展示 AI 如何協助實際的程式開發流程
- 學生可以看到從需求到實作的完整過程
- 配合「反向解構」教學法，讓學生觀察 AI 如何組織和撰寫程式碼

## 注意事項
需要付費訂閱（Max、Pro、Team 或 Enterprise）才能使用此功能。

## 範例大綱

以下是 2 個實際操作範例，涵蓋 Bug 修復和程式碼重構。每個範例都包含詳細的操作步驟和可直接複製貼上使用的 Prompt 對話範例：

### 📌 [範例 1：Bug 修復與除錯](./範例1-Bug修復與除錯.md)

**目標**：使用 Claude Code 來診斷和修復程式碼中的錯誤

**教學重點**：
- 展示 AI 如何協助診斷程式錯誤
- 學習如何描述問題和提供上下文
- 了解除錯的最佳實踐

**快速預覽**：
1. 描述錯誤並請求 AI 診斷問題
2. 觀察 AI 如何分析錯誤原因
3. 執行系統化檢查，找出相關問題
4. 學習有效的問題描述技巧

### 📌 [範例 2：程式碼重構與優化](./範例2-程式碼重構與優化.md)

**目標**：重構和優化現有程式碼，提升可讀性、維護性和效能

**教學重點**：
- 學習程式碼重構的流程和方法
- 了解如何識別可以優化的程式碼模式
- 掌握程式碼品質改善的技巧

**快速預覽**：
1. 找出專案中的重複程式碼
2. 建立通用的函式來取代重複邏輯
3. 分析效能瓶頸並優化
4. 為重構後的程式碼撰寫測試
