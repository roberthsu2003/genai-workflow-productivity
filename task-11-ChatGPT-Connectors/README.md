# 任務 11：使用 ChatGPT Connectors 整合外部資料來源

## 🎯 工作情境
需要在 ChatGPT 中直接存取和整理 Google Drive、GitHub、SharePoint 等外部資料來源，用於報告分析、文件整理或研究，避免手動下載和複製貼上。

## 🤖 使用的 AI 工具
- **ChatGPT**（Plus/Pro 方案支援 Connectors，免費版功能有限）
- **Connectors/Apps**（Google Drive、GitHub、SharePoint 等外部工具整合）
- **外部資料來源**（Google Drive、GitHub Repositories、SharePoint 等）

## 💻 AI 負責的工作（80%）
- 自動從外部資料來源讀取和檢索文件
- 分析多個資料來源的內容
- 統整不同格式的文件（PDF、Word、程式碼等）
- 生成整合分析和摘要報告
- 提取關鍵資訊並生成圖表建議

## 👤 你負責的工作（20%）
- 選擇和配置外部 Connector（一次性設定）
- 決定哪些資料來源和資料夾重要
- 定義分析需求和報告格式
- 審閱 AI 整合的內容準確性
- 確認資料授權和隱私設定

## 📝 實作步驟
1. **確認 ChatGPT 方案**：檢查 ChatGPT 帳號方案（Connectors 主要在 Plus/Pro 方案可用）
2. **設定 Connector（不寫程式）**：在 ChatGPT 設定 → Apps & Connectors 中，連接需要的外部工具
3. **測試連接**：在 ChatGPT 中詢問，確認 ChatGPT 可以正確存取外部資料來源
4. **整合資料任務**：告訴 ChatGPT 整合需求，AI 會自動從多個來源收集和整理資料
5. **審閱和調整**：檢查 AI 整合的內容，確認資料來源引用的準確性
6. **輸出和分享**：將整合結果匯出為文件或直接分享

## 📄 產出結果
- **整合的分析報告**（從多個外部來源自動整合）
- **資料來源引用清單**（包含每個資訊的來源）
- **自動化的資料檢索流程**（可重複使用）

## 💡 實際應用範例

**範例 1：研究報告整合**
- **資料來源**：Google Drive（研究論文）、GitHub（程式碼和數據）、SharePoint（公司內部資料）
- **產出**：包含多個來源引用的完整研究報告

**範例 2：程式碼文件分析**
- **資料來源**：GitHub Repositories（多個專案的程式碼和文件）
- **產出**：技術文件摘要和程式碼結構分析

**範例 3：客戶資料整合**
- **資料來源**：Google Drive（客戶資料）、SharePoint（內部記錄）
- **產出**：客戶關係管理報告和統計分析

## ⚙️ ChatGPT Connectors 設定說明（免程式）

**Google Drive Connector**（Plus/Pro 方案）
- 在 ChatGPT 設定中選擇「Google Drive」Connector
- 點選授權按鈕，登入 Google 帳號

**GitHub Connector**（Plus/Pro 方案）
- 在設定中選擇「GitHub」Connector
- 登入 GitHub 帳號並授權

**SharePoint Connector**（企業版）
- 在設定中選擇「SharePoint」Connector
- 輸入 SharePoint 位址和認證資訊
