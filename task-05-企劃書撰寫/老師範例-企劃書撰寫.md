# 任務 5：企劃書撰寫（老師範例）

> **本範例示範如何使用 AI 工具（ChatGPT/Claude、Perplexity、Google Docs、Gemini）協助撰寫完整的企劃書**

---

## 📋 企劃書基本資訊

- **企劃主題**：校園數位學習平台開發計畫
- **企劃目標**：建立一個整合線上學習資源的平台，提升學生學習效率
- **企劃期間**：6 個月
- **預算範圍**：新台幣 50 萬元
- **目標受眾**：學校管理階層、投資人
- **製作時間**：5 天
- **製作人**：張專案經理（使用 AI 工具協助）

---

## 🎯 專案定義與架構生成

### 步驟 1：專案定義

**使用 ChatGPT 定義專案範圍**

**提示詞：**
```
我需要撰寫一份專案企劃書，請協助我定義專案範圍：

專案目標：建立校園數位學習平台
預算範圍：新台幣 50 萬元
執行期間：6 個月
目標受眾：學校管理階層、投資人

請幫我：
1. 明確專案目標和範圍
2. 列出專案關鍵成功因素
3. 識別主要風險點
4. 建議專案架構
```

**AI 生成的專案定義：**

**專案目標：**
- 建立一個整合線上學習資源的平台
- 提供個人化學習路徑
- 支援多媒體學習內容
- 整合學習進度追蹤功能

**專案範圍：**
- 平台開發（前端 + 後端）
- 內容管理系統
- 使用者管理系統
- 數據分析儀表板

**關鍵成功因素：**
1. 使用者體驗設計優良
2. 內容品質高
3. 技術穩定可靠
4. 行銷推廣到位

**主要風險點：**
1. 技術開發延遲
2. 預算超支
3. 使用者接受度低
4. 競爭對手威脅

---

### 步驟 2：生成企劃書架構

**提示詞：**
```
請為我生成一份完整的企劃書大綱，主題是「校園數位學習平台開發計畫」。
需要包含：
1. 執行摘要
2. 專案背景與目標

### 企劃書架構（20-30 頁）

**第 1 章：執行摘要**（2 頁）
- 專案概述
- 核心價值主張
- 關鍵數字（預算、時程、預期成效）

**第 2 章：專案背景與目標**（3 頁）
- 專案背景
- 問題定義
- 專案目標
- 目標受眾

**第 3 章：市場分析**（4 頁）
- 市場現況
- 競爭者分析
- 目標市場
- 市場機會點

**第 4 章：專案內容**（5 頁）
- 功能需求
- 技術架構
- 系統設計
- 使用者介面設計

**第 5 章：執行計畫**（4 頁）
- 時程規劃
- 資源配置
- 團隊組織
- 里程碑設定

**第 6 章：預算規劃**（3 頁）
- 預算細項
- 資金來源
- 成本控制

**第 7 章：風險評估**（3 頁）
- 風險識別
- 風險分析
- 應對策略

**第 8 章：預期成效**（3 頁）
- 量化指標
- 質化效益
- 投資回報分析

**第 9 章：結論**（1 頁）
- 專案價值
- 呼籲行動

## 📊 市場分析（使用 Perplexity）

**查詢主題：** 台灣數位學習平台市場現況

**使用 Perplexity 查詢**

**查詢提示：**
```

1. 市場規模和成長率

4. 市場趨勢和機會點

  - 平台A（市占率 30%）
  - 平台B（市占率 25%）
  - 平台C（市占率 20%）
- 使用者行為：
  - 78% 的學生使用數位學習平台
  - 平均每週使用時間：5-8 小時
  - 最受歡迎功能：個人化學習路徑、影片教學
- 市場趨勢：
  - AI 輔助學習崛起
  - 行動裝置使用率增加
  - 個人化學習需求上升
- 政策支持：
  - 教育部推動數位學習政策
  - 補助學校建置數位學習環境

**數據來源：**
- 2024 台灣數位學習市場研究報告
- 教育部統計資料
- 產業分析公司市場調查

### 步驟 4：使用 AI 整理市場分析

**提示詞：**
```
請根據以下市場數據，撰寫一份專業的市場分析段落：

[貼上 Perplexity 查詢結果]

要求：
1. 分析市場現況和趨勢
2. 比較主要競爭者
3. 識別市場機會點
4. 說明目標市場定位

使用專業但易懂的商業語言，適合管理階層閱讀。
```

**AI 生成的市場分析：**

#### 市場分析

**市場現況**

台灣數位學習市場持續成長，2024 年市場規模達到新台幣 150 億元，過去 3 年年均成長率為 15%。主要成長動因包括：
- 數位化教育政策推動
- 行動裝置普及
- AI 技術應用
- 個人化學習需求增加

**競爭者分析**

目前市場主要競爭者包括：
- **平台A**（市占率 30%）：功能完整但價格較高
- **平台B**（市占率 25%）：使用者體驗佳但內容較少
- **平台C**（市占率 20%）：內容豐富但介面較舊

**市場機會點**

1. **個人化學習**：78% 的學生希望有個人化學習路徑
2. **行動學習**：行動裝置使用率持續上升
3. **AI 輔助**：AI 技術可提供更智能的學習體驗
4. **校園整合**：整合校園系統的平台較少

**目標市場定位**

本專案定位為「校園專屬數位學習平台」，強調：
- 與校園系統整合
- 個人化學習路徑
- 優質內容提供
- 合理價格（針對校園優惠）

## 💰 預算規劃（使用 ChatGPT）

### 步驟 6：生成預算規劃

**提示詞：**
```
請為校園數位學習平台開發計畫生成預算規劃：

總預算：新台幣 50 萬元
項目包括：
1. 系統開發（前端、後端）
2. 內容製作
3. 行銷推廣
4. 營運維護
5. 其他費用

請提供詳細的預算分配和說明。
```

**AI 生成的預算規劃：**

#### 預算規劃

**總預算：新台幣 50 萬元**

| 項目 | 預算 | 說明 |
|------|------|------|
| 系統開發 | 25 萬元 | 前端開發（10萬）、後端開發（10萬）、系統整合（5萬） |
| 內容製作 | 10 萬元 | 課程內容製作、影片拍攝、教材設計 |
| 行銷推廣 | 8 萬元 | 宣傳活動、使用手冊、訓練課程 |
| 營運維護 | 5 萬元 | 主機租賃、技術支援、內容更新 |
| 其他費用 | 2 萬元 | 預備金、雜項支出 |

**資金來源：**
- 學校預算：30 萬元（60%）
- 外部贊助：20 萬元（40%）

## ⚠️ 風險評估（使用 ChatGPT）

### 步驟 7：識別與評估風險

**提示詞：**
```
請為校園數位學習平台開發計畫進行風險評估：

請識別：
1. 技術風險（開發延遲、技術難題）
2. 預算風險（成本超支）
3. 市場風險（使用者接受度）
4. 營運風險（系統穩定性）

對於每個風險，請提供：
- 風險描述
- 發生機率（高/中/低）
- 影響程度（高/中/低）
- 應對策略
```

**AI 生成的風險評估：**

#### 風險評估

**1. 技術開發延遲**
- **風險描述**：系統開發可能因技術難題而延遲
- **發生機率**：中
- **影響程度**：高
- **應對策略**：
  - 預留緩衝時間（20%）
  - 聘請有經驗的開發團隊
  - 採用成熟的技術框架

**2. 預算超支**
- **風險描述**：實際成本可能超過預算
- **發生機率**：中
- **影響程度**：中
- **應對策略**：
  - 預留預備金（5%）
  - 分階段執行，控制成本
  - 定期檢視預算使用狀況

**3. 使用者接受度低**
- **風險描述**：師生可能不願意使用新平台
- **發生機率**：低
- **影響程度**：高
- **應對策略**：
  - 早期邀請測試使用者
  - 提供使用訓練
  - 收集意見並快速改善

## 🤖 AI 工具使用記錄

### 使用流程

**第 1-2 天：專案定義與架構**
1. 使用 ChatGPT 定義專案範圍（2 小時）
2. 生成企劃書架構（1 小時）
3. 調整架構符合需求（1 小時）

**第 3 天：市場分析**
4. 使用 Perplexity 查詢市場數據（2 小時）
5. 使用 ChatGPT 整理分析（2 小時）

**第 4 天：內容撰寫**
6. 逐章使用 ChatGPT 撰寫內容（6 小時）
7. 人工調整和優化（2 小時）

**第 5 天：預算與風險**
8. 生成預算規劃（1 小時）
9. 進行風險評估（1 小時）
10. 最終整合與檢查（3 小時）

### AI 工具優缺點

**優點：**
- ✅ 大幅節省時間：從 2 週縮短至 5 天
- ✅ 架構完整：AI 生成的架構邏輯清晰
- ✅ 市場數據快速：Perplexity 快速提供最新數據
- ✅ 內容豐富：AI 能提供多角度的分析

**需要改進的地方：**
- ⚠️ 內容需要人工調整和驗證
- ⚠️ 數據準確性需要多重驗證
- ⚠️ 預算和風險需要根據實際情況調整
