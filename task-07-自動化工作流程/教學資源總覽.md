## 🎯 教學流程建議

### 第一階段：介紹與示範（30-40 分鐘）

1. **介紹任務目標**（5 分鐘）
   - 說明為什麼要學習自動化工作流程
   - 介紹 AI 工具在其中的作用
   - 說明自動化帶來的效益

2. **展示老師範例**（15-20 分鐘）
   - 使用「老師範例-自動化工作流程.md」
   - 重點說明：
     - 系統架構設計
     - 表單設計原則
     - 自動化腳本的功能
     - API 整合方式
     - 觸發器設定

3. **AI 工具示範**（10-15 分鐘）
   - 現場示範 Google Apps Script 的基本操作
   - 展示如何使用 ChatGPT API 進行分析
   - 說明 API 使用成本和免費替代方案
   - 示範如何設定觸發器

### 第二階段：學生練習（60-90 分鐘）

1. **提供練習材料**（10 分鐘）
   - 提供一個實際的使用情境（如課程意見回饋、活動報名等）
   - 讓學生設計自己的自動化流程
   - 說明練習要求和標準

2. **學生實作**（40-60 分鐘）
   - 使用「學生實作範例-模板.md」
   - 要求學生完成：
     - 表單設計
     - 資料儲存結構設計
     - 自動化腳本撰寫（或規劃）
     - 觸發器設定
     - 測試和優化
   - 鼓勵學生使用 AI 工具輔助
   - 提醒需要人工檢視和修正

3. **展示學生範例**（10-20 分鐘）
   - 展示「學生實作範例-範例.md」
   - 說明完成標準和要求
   - 解答常見問題

### 第三階段：檢討與改進（20-30 分鐘）

1. **同儕互評**（10-15 分鐘）
   - 學生交換作品
   - 對照評分標準進行互評
   - 提供改進建議

2. **老師講評**（10-15 分鐘）
   - 指出常見錯誤
   - 強調重點技巧
   - 分享優秀作品
   - 說明最佳實踐

## 💡 常見問題與解答

### Q1: 如果學生沒有 Google 帳號怎麼辦？

**A:** 
- 可以使用其他表單工具（Microsoft Forms、Typeform 等）
- 或使用學校提供的 Google Workspace 教育帳號
- 也可以使用其他自動化平台（Zapier、Microsoft Power Automate 等）

### Q2: API 費用怎麼辦？

**A:** 
- 可以使用免費方案：手動批次使用 ChatGPT 網頁版分析
- 或使用 Google Gemini API（有免費額度）
- 示範時可以只展示如何整合 API，不需要實際執行
- 強調免費替代方案的重要性

### Q3: 學生不會寫程式怎麼辦？

**A:** 
- 可以簡化需求，重點在流程設計而非完整實作
- 使用視覺化自動化工具（Zapier、Microsoft Power Automate）
- 提供程式碼範例，讓學生理解概念
- 強調學習程式設計的重要性，鼓勵自主學習

### Q4: 如何評估學生是否真正理解？

**A:** 
- 要求學生清楚說明自動化流程的設計邏輯
- 要求學生記錄遇到的問題和解決方式
- 在檢討階段詢問學生的思考過程
- 評估學生能否改進和優化流程

### Q5: 可以在沒有網路的情況下教學嗎？

**A:** 
- 可以使用離線版本的範例檔案
- 示範如何設計流程架構
- 說明程式碼邏輯和概念
- API 整合部分可以稍後在有網路時示範
