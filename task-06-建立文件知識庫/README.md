# 任務 6：建立文件知識庫

## 🎯 工作情境
公司有大量 SOP、技術文件、政策手冊，新進員工或遇到問題時，需要快速找到相關資訊並理解。

## 🤖 使用的 AI 工具
- **Ollama**（本地 RAG，資料不上雲）
- **NotebookLM**（Google 文件知識庫）
- **Claude**（文件摘要和問答）

## 💻 AI 負責的工作（80%）
- 自動索引所有文件內容
- 理解文件語意和關聯性
- 根據問題檢索相關段落
- 生成摘要和重點整理
- 回答文件相關問題

## 👤 你負責的工作（20%）
- 收集和整理公司文件
- 確認文件版本和有效性
- 驗證 AI 回答的準確性
- 標記機密文件（避免上傳）
- 持續更新知識庫

## 📝 實作步驟
1. **文件收集**：將公司文件（PDF、Word、Google Docs）整理到一個資料夾
2. **建立知識庫**：
   - **方法 A（雲端）**：將文件上傳到 NotebookLM
   - **方法 B（本地）**：使用 Ollama + RAG 工具建立本地知識庫
3. **測試問答**：詢問常見問題，確認 AI 能正確檢索和回答
4. **建立使用指南**：撰寫如何使用知識庫的 SOP
5. **持續維護**：定期更新文件，確保知識庫的時效性

## 📄 產出結果
- 可查詢的公司文件知識庫
- 常見問題 Q&A 文件
- 知識庫使用手冊
