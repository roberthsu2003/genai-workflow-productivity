### 步驟一：確認 MCP 連接狀態

**執行動作：**
在 Claude Desktop 中詢問：
```
請確認目前有哪些 MCP Connector 已連接？
```

**Claude 回應：**
- ✅ Filesystem MCP（本地檔案系統）- 已連接
- ✅ Google Drive MCP - 已連接
- ⚠️ Notion MCP - 未連接（本次任務不需要）

### 步驟二：指定資料來源並要求整合

**執行動作：**
在 Claude Desktop 中輸入以下指令：
```
請幫我整合以下資料來源，生成本週工作週報：

資料來源：
1. Google Drive 資料夾「會議紀錄/2025-01/」中的所有檔案
2. 本地檔案系統路徑「~/Documents/專案進度/本週/」中的所有 Markdown 檔案
3. 本地檔案系統路徑「~/Documents/待辦事項/本週任務.txt」

請將這些資料整理成一份結構化的週報，包含：
- 本週完成事項（依時間順序）
- 進行中的工作（含進度百分比）
- 待辦事項（依優先順序）
- 下週計畫
- 重要決議事項（從會議紀錄中提取）

格式使用 Markdown，並加上適當的標題和表格。
```

### 步驟三：Claude 自動整合資料

**Claude 執行過程：**
1. 讀取 Google Drive 中的 3 個會議紀錄檔案

3. 讀取本地檔案系統中的待辦事項清單
4. 分析所有資料內容
5. 提取關鍵資訊並分類

## 🤖 AI 工具使用說明

本週報使用以下 AI 工具協助製作：

### 1. Claude Desktop
- **功能**：整合 MCP Connector，自動讀取多個資料來源
- **使用方式**：透過自然語言指令，要求 Claude 讀取和整合資料
- **優點**：一次整合多個來源，節省手動切換和複製貼上的時間

### 2. MCP Connector（Model Context Protocol）
- **使用的 Connector**：
  - **Filesystem MCP**：讀取本地檔案系統
  - **Google Drive MCP**：讀取 Google Drive 中的檔案
- **設定方式**：
  - 在 Claude Desktop 設定中啟用對應的 MCP Connector
  - 授權 Google Drive 存取權限
  - 指定本地檔案系統可存取的資料夾路徑
- **優點**：免寫程式，透過設定即可連接各種資料來源

### 3. 使用的 Prompt 範例
```
請幫我整合以下資料來源，生成本週工作週報：

1. Google Drive 資料夾「會議紀錄/2025-01/」中的所有檔案
```

## 💡 使用心得與建議

### AI 工具的優點
1. **節省時間**：原本需要 1-2 小時手動整合的資料，現在只需 10-15 分鐘
2. **減少錯誤**：AI 自動提取和分類，避免人工複製貼上時的遺漏或錯誤
3. **結構化輸出**：自動生成格式化的 Markdown 文件，可以直接使用
4. **跨平台整合**：一次整合多個資料來源，無需手動切換應用程式

### 需要注意的事項
1. **資料準確性**：AI 產出的內容仍需要人工檢視和確認
2. **資料來源品質**：原始資料的完整性和格式會影響 AI 的分析結果
3. **隱私安全**：確保 MCP Connector 設定的存取權限符合資料安全政策
4. **Prompt 精確度**：給予清楚明確的指令，可以獲得更準確的結果

### 改進建議
1. **建立範本**：可以將常用的 Prompt 儲存為範本，方便重複使用
2. **定期更新**：建議每週固定時間執行整合任務，保持資料即時性
3. **格式優化**：根據團隊需求調整輸出格式，讓週報更符合使用習慣
