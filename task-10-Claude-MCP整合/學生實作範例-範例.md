---

## 📋 任務資訊

- **任務名稱**：數位行銷課程專案進度週報自動生成
- **日期**：2025年1月20日（星期一）
- **任務類型**：跨應用程式資料整合
- **使用的 MCP Connector**：本地檔案系統、Google Drive
- **執行人**：張同學

### 資料來源
- **Google Drive**：小組會議紀錄（2 個檔案在「課程專案/會議紀錄/2025-01/」資料夾）
- **本地檔案系統**：個人工作進度報告（1 個 Markdown 檔案在「~/Documents/專案/進度報告.md」）
- **本地檔案系統**：待辦事項清單（1 個文字檔在「~/Documents/專案/待辦事項.txt」）

---

## 📝 任務目標

整合本週從多個來源收集的專案資料，生成一份完整的進度週報，包含：
1. 本週完成事項
2. 進行中的工作
3. 遇到的困難
4. 下週計畫
5. 需要協助的事項

---

## 🤖 AI 工具使用流程

### 步驟一：確認 MCP 連接狀態

**執行動作：**
在 Claude Desktop 中詢問：
```
請確認目前有哪些 MCP Connector 已連接？
```

**Claude 回應：**
- ✅ Filesystem MCP（本地檔案系統）- 已連接
- ✅ Google Drive MCP - 已連接
- ⚠️ Notion MCP - 未連接（本次任務不需要）

### 步驟二：指定資料來源並要求整合

**執行動作：**
在 Claude Desktop 中輸入以下指令：
```
請幫我整合以下資料來源，生成專案進度週報：

資料來源：
1. Google Drive 資料夾「課程專案/會議紀錄/2025-01/」中的所有檔案
2. 本地檔案系統路徑「~/Documents/專案/進度報告.md」
3. 本地檔案系統路徑「~/Documents/專案/待辦事項.txt」

請將這些資料整理成一份結構化的週報，包含：
- 本週完成事項（依時間順序）
- 進行中的工作（含進度百分比）
- 遇到的困難
- 下週計畫
- 需要協助的事項

格式使用 Markdown，並加上適當的標題和表格。
```

### 步驟三：Claude 自動整合資料

**Claude 執行過程：**
1. 讀取 Google Drive 中的 2 個會議紀錄檔案（「2025-01-15-週會.md」和「2025-01-18-討論會.md」）
2. 讀取本地檔案系統中的進度報告（「進度報告.md」）
3. 讀取本地檔案系統中的待辦事項清單（「待辦事項.txt」）
4. 分析所有資料內容，提取關鍵資訊
5. 分類整理成結構化的格式
6. 生成 Markdown 格式的週報

## 🤖 AI 工具使用記錄

**使用的 AI 工具：**
- ✅ Claude Desktop
- ✅ MCP Connector（本地檔案系統）
- ✅ MCP Connector（Google Drive）

**MCP Connector 設定過程：**
1. 步驟 1：在 Claude Desktop 設定中啟用 Filesystem MCP
   - 指定可存取的資料夾路徑：`~/Documents/專案/`
   
2. 步驟 2：在 Claude Desktop 設定中啟用 Google Drive MCP
   - 點選授權按鈕
   - 登入 Google 帳號
   - 授予 Claude Desktop 存取 Google Drive 的權限

3. 步驟 3：測試連接
   - 詢問 Claude 是否可以讀取本地檔案
   - 詢問 Claude 是否可以讀取 Google Drive 檔案
   - 確認連接成功

**使用的 Prompt：**
```
請幫我整合以下資料來源，生成專案進度週報：

資料來源：
1. Google Drive 資料夾「課程專案/會議紀錄/2025-01/」中的所有檔案
2. 本地檔案系統路徑「~/Documents/專案/進度報告.md」
3. 本地檔案系統路徑「~/Documents/專案/待辦事項.txt」

請將這些資料整理成一份結構化的週報，包含：
- 本週完成事項（依時間順序）
- 進行中的工作（含進度百分比）
- 遇到的困難
- 下週計畫
- 需要協助的事項

格式使用 Markdown，並加上適當的標題和表格。
```

**AI 工具的優缺點：**
- **優點**：
  - 節省時間：原本需要 30-40 分鐘手動整合的資料，現在只需 5-10 分鐘
  - 自動提取關鍵資訊：AI 能從多個檔案中自動提取相關資訊，不用逐個閱讀
  - 結構化輸出：自動生成格式化的 Markdown 文件，可以直接使用
  - 跨平台整合：一次整合多個資料來源，無需手動切換應用程式和複製貼上

- **需要改進的地方**：
  - 資料準確性需要人工確認：AI 可能會遺漏一些細節或誤解某些資訊
  - 格式需要微調：生成的格式有時需要根據需求稍微調整
  - 需要清楚的資料結構：如果原始資料格式不一致，AI 整合的結果可能不如預期

## 💡 使用心得與反思

### 學習收穫
- **MCP 功能強大**：透過 MCP Connector，Claude 可以直接讀取多個來源的資料，這個功能非常實用
- **Prompt 設計重要**：清楚的 Prompt 可以讓 AI 更準確地執行任務，需要練習如何寫好 Prompt
- **節省大量時間**：原本需要花很多時間手動整合的資料，現在可以快速完成

### 遇到的困難與解決方法
- **困難 1**：第一次使用時不知道如何設定 MCP Connector
  - **解決方法**：參考教學資源總覽中的說明文件，並在 Claude Desktop 設定中逐步操作

- **困難 2**：第一次整合時，AI 產生的格式不太符合需求
  - **解決方法**：在 Prompt 中更詳細地說明需要的格式，並在 AI 產出後進行微調

### 改進建議
- **建立 Prompt 範本**：可以將常用的 Prompt 儲存為範本，方便重複使用
- **資料整理**：建議在整合前先整理原始資料的格式，讓 AI 能更準確地分析
- **定期使用**：每週固定時間執行整合任務，保持資料即時性
