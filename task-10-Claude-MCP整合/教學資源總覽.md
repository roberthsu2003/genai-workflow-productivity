## 🎯 教學流程建議

### 第一階段：介紹與示範（30-40 分鐘）

1. **介紹任務目標**（5 分鐘）
   - 說明為什麼要學習 Claude MCP 整合
   - 介紹 MCP（Model Context Protocol）的基本概念
   - 說明 AI 工具在跨應用程式整合中的作用

2. **展示老師範例**（15-20 分鐘）
   - 使用「老師範例-Claude-MCP整合.md」
   - 重點說明：
     - MCP Connector 的設定方式
     - 如何指定資料來源
     - Prompt 的設計技巧
     - 整合結果的格式和內容
     - AI 工具使用的優缺點

3. **Claude Desktop 與 MCP 示範**（10-15 分鐘）
   - 現場示範 Claude Desktop 的安裝和基本操作
   - 示範如何設定 MCP Connector（本地檔案系統、Google Drive）
   - 展示如何透過自然語言指令要求 Claude 整合資料
   - 說明 MCP 的限制和注意事項

### 第二階段：學生練習（50-70 分鐘）

1. **提供練習材料**（5 分鐘）
   - 提供多個資料來源（Google Drive 檔案、本地檔案等）
   - 或讓學生準備自己的資料來源

2. **學生實作**（40-60 分鐘）
   - 使用「學生實作範例-模板.md」
   - 要求學生：
     - 設定 MCP Connector（至少連接本地檔案系統）
     - 設計 Prompt 要求 Claude 整合資料
     - 記錄整個過程和結果
     - 反思 AI 工具的優缺點
   - 提醒需要人工檢視和修正

3. **展示學生範例**（5-10 分鐘）
   - 展示「學生實作範例-範例.md」
   - 說明完成標準和要求

### 第三階段：檢討與改進（20-30 分鐘）

1. **同儕互評**（10-15 分鐘）
   - 學生交換作品
   - 對照評分標準進行互評
   - 提供改進建議

2. **老師講評**（10-15 分鐘）
   - 指出常見錯誤
   - 強調重點技巧（Prompt 設計、MCP 設定等）
   - 分享優秀作品

## 💡 常見問題與解答

### Q1: 如果學生沒有 Claude Desktop 帳號怎麼辦？

**A:** 
- Claude Desktop 可以免費使用（需註冊 Anthropic 帳號）
- 免費版已包含基本的 MCP 功能
- 老師可以協助學生註冊帳號，或使用示範帳號進行教學

### Q2: 學生應該完全依賴 AI 工具嗎？

**A:** 
- 不應該。強調 AI 是輔助工具
- 學生需要學習如何設計好的 Prompt
- 需要檢視和修正 AI 產生的內容
- 培養判斷能力和批判思考

### Q3: MCP Connector 設定複雜嗎？

**A:** 
- 本地檔案系統 MCP 設定很簡單，只需指定資料夾路徑
- Google Drive MCP 需要授權，但過程很直觀
- 其他 MCP Connector 的設定方式類似，但可能需要額外的 Token 或授權
- 建議先從簡單的本地檔案系統開始練習

### Q4: 可以在沒有網路的情況下教學嗎？

**A:** 
- Claude Desktop 需要網路連線才能使用
- 可以使用離線版本的範例檔案進行說明
- 實際操作需要在有網路的情況下進行
- 建議確保課堂有穩定的網路連線

### Q5: 哪些資料來源可以透過 MCP 連接？

**A:** 
- 常見的 MCP Connector 包括：
  - 本地檔案系統（Filesystem MCP）
  - Google Drive
  - Notion
  - Slack（部分版本）
  - Canva
  - 資料庫（需要額外設定）
- 建議參考官方文件或教學資源連結了解最新的 MCP Connector 清單
